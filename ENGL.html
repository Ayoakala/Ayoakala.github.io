Tidying our Data¶

Data tiding is an essential part of Data Science; many Data Scientists report that most of their time that they spend on projects is composed of mainly structuring their data into a form that is easier to digest when they perform Exploratory Data Analysis (or EDA) and even Machine Learning. We must be careful and thoughtful of how to structure our data. Here a few points on what we thought was important and worthy of addressing:

  • In our previous output, we see that there is a mismatch in one of our attributes (or columns), the race attribute is flipped with the sex attribute. We will first make a new column in our Data frame to temporarily store the old attribute values of our current "sex" attribute into "race_new", We do the same for the "race" attribute. We then assign "race_new" and "sex_new" to their corresponding attributes.

  • We have now created a small problem for ourselves, we now have 2 redundant attributes which contain repeated data points. We will drop these tables to create a more parsimonious dataframe (or table)

  • Taking a glance through the whole data set we spotted some nulled-out values within our "Location 1" attribute which is basically the attribute that harbors our latitude and longitude values. This would interfere with some of our analysis because some of the functions that we use have undefined results on null data. We will remove those instances from our dataset.

  • To broadcast the geographic location on our folium map we must get the latitude and longitude of the location. In our previous output out latitude and longitude values are concatenated together as a string which would not be a proper form to pass into the map. The proper form would be 2 floating point type values. We outline one way of doing this by using the split function (to delimit our attribute) and the astype function (to parse the string value into a floating-point type).

In [3]:
# tidy up data to make EDA and visualization easier
arrest_table["race_new"] = arrest_table["sex"]
arrest_table["sex_new"] = arrest_table["race"]
arrest_table["race"] = arrest_table["race_new"]
arrest_table["sex"] = arrest_table["sex_new"]

# dropping redundant attributes
arrest_table = arrest_table.drop('race_new', 1)
arrest_table = arrest_table.drop('sex_new', 1)

# remove null locations
arrest_table = arrest_table[pandas.notnull(arrest_table["Location 1"])]

# delimiting our location attribute and parsing the attributes to a floating point value
arrest_table["lat"], arrest_table["long"] = arrest_table["Location 1"].str.split(",").str
arrest_table["lat"] = arrest_table["lat"].str.replace("(", "").astype(float)
arrest_table["long"] = arrest_table["long"].str.replace(")", "").astype(float)

Exploratory Data Analysis¶

In this section, we explore visualizations of different indicators through our cleaner dataframes. The goal of this section is to view our data on a map that gives us a good outlook on crime in Baltimore rather than looking though a repetitive and unintuitive dataframe to understand trends. After looking at our data we decided the best mode of operation is to choose a random sample of 500 different crimes. We did this because our dataset has over 7000 examples and would be computationally inefficient to process all the data especially for educational purposes. We will then partition our dataframe into 9 distinct dataframes based on the district in which the crime happened in Baltimore and then gather general statistics to automatically figure out the clusters of districts (we dont have data on where the district is).

In [4]:

crime_stats = {}
for d in range(len(districts)):
    for i in districts[d].iterrows():
        elements = i[1]
        n = elements[11]
        crime = elements[7]
        if n not in crime_stats:
            crime_stats[n] = {} 
        if crime not in crime_stats[n]:
            crime_stats[n][crime] = 0
        crime_stats[n][crime] += 1
       
# Select the top 5 offenses and create and HTML representation of our findings
top_five = {}
for d in crime_stats.keys():
    curr_crime = crime_stats[d]
    total_dist = sum(crime_stats[d].values())
    top_five[d] = [str(k + ': ' + str(curr_crime[k]) + " <b>(" + str(int(curr_crime[k]/total_dist*100)) + "%)</b>") for k in sorted(curr_crime, key=curr_crime.get, reverse=True)][:5]

for s in sample:
    for i in s.iterrows():
        element = i[1]

        age = element[1]
        sex = element[2]
        race = element[3]

        description  = "<b>Age:</b>  %s</br><b>Sex:</b> %s</br><b>Race:</b> %s</br><b>Arrest Date:</b> %s" % (age,sex,race,arrest_date)
        folium.CircleMarker(
            location=coords,
            radius=8,
            popup= description,
            color='black',
            fill = True,
            fill_color=sex_legend[sex],
            fill_opacity= .65,
            weight = .9,
            icon=folium.Icon(color=sex_legend[sex],icon="remove-sign")).add_to(map_osm)

District Overlay¶

We will now make the use of the supervised method of figuring out our district cluster by averaging the different latitudes and longitudes. We will now make use of those predicted district clusters and project them or overlay them on our map along with the summary statistics of that particular district.

In [8]:
# District overlay
colors = {0:'blue', 1:'red', 2:'green', 3:'purple', 4:'darkpurple', 5:'black', 6:'lightgray', 7:'orange', 8:'lightgreen'}

for d in range(len(districts)-1):
    temp = districts[d].head(1)
    name = temp['district'].values[0]
    description = "<b>"+ name + "</b></br>" + "</br>".join(top_five[name])
    coords = [np.average(districts[d]['lat']),np.average(districts[d]['long'])]
    folium.Marker(coords,
                  popup=description,
                  icon=folium.Icon(
                      color=colors[d])).add_to(map_osm)
